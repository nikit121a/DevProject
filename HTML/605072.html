                                Sasha (DevExpress Support) 02.12.2018

                                Hello Nik,

                                Thank you for the code snippet. Your code is correct. In fact, it's not necessary to add an appointment to the Scheduler storage manually and the GetSourceObject method should return the corresponding object. Most likely, it's null in your case because
                                the LinkOccurrencesWithPatterns method is called between the Storage.BeginUpdate/Storage.EndUpdate method calls. Thus, the object is not created until the EndUpdate method is called.
                                I suggest that you modify the Import and LinkOccurrencesWithPatterns methods as follows:

                                [C#]
                                            //LinkOccurrencesWithPatterns
            Appointment ex = null;
            if (exceptionType == AppointmentType.DeletedOccurrence) {
                ex = CreateDeletedOccurrence(srcPattern, occurrenceEvent);
            }
            else {
                ex = CreateChangedOccurrence(srcPattern, occurrenceEvent);
            }
            return ex;
                                [C#]
                                        public void Import(IList<Event> events) {
            Dictionary<string, Appointment> patternHash = new Dictionary<string, Appointment>();
            Dictionary<string, Event> occurrenceHash = new Dictionary<string, Event>();
            Appointment app = null;
            Storage.BeginUpdate();
            try {
                 //......
                }
                app = LinkOccurrencesWithPatterns(occurrenceHash, patternHash);
            } finally {
                Storage.EndUpdate();
                MyAppointment exApp = app.GetSourceObject(Storage) as MyAppointment;
                if (exApp != null) {
                   exApp.SaveToDataBase();
                }
            }
        }

                                Please try this approach and let me know if it helps.

                                • Svetlana (DevExpress Support) 03.22.2018

                                  Hi, 

                                  Now, I have reproduced this behavior. The Google Calendar Service passes only information specific to a changed occurrence. Thus, as you correctly mentioned, the patternHash Dictionary is empty. That is why it is necessary to find a corresponding pattern appointment in the SchedulerStorage.Appointments collection. I should note that in the UpdateFromGoogleCalendar method, we clear the Appointments collection before calling the CalendarImporter.Import method. Thus, since you do not wish to reload all appointments each time data is synchronized between your app and the Google calendar, it is necessary to comment out this line.
                                  I should note that in the Import method, we save the id of an imported appointment to the eventId custom field. Thus, you will be able to find a pattern appointment by traversing through the Appointments collection and checking if a value returned from an appointment's CustomFields["eventId"] property equals the patternId field value in the LinkOccurrencesWithPatterns method.

                                  I look forward to your results once you try this approach.
