                                Hello Mustafa,

                                Thank you for the video. To accomplish this task, perform the following steps.
                                1. Mark each cell by specifying its Cell.CssClass property:

                                [C#]
                                grid.Styles.Cell.CssClass = "mycell";

                                2. Set the SettingsEditing.Mode property to the "PopupEditForm" value:

                                [C#]
                                grid.SettingsEditing.Mode = GridViewEditingMode.PopupEditForm;

                                3. Handle the ASPxGridView.HtmlDataCellPrepared event in the following manner:

                                [C#]
                                    protected void ASPxGridView1_HtmlDataCellPrepared(object sender, DevExpress.Web.ASPxGridViewTableDataCellEventArgs e) {
        e.Cell.Attributes.Add("ondblclick", String.Format("onCellDoubleClick({0}, '{1}')", e.VisibleIndex, e.DataColumn.FieldName));
        e.Cell.Attributes.Add("onclick", String.Format("onCellClick({0}, '{1}')", e.VisibleIndex, e.DataColumn.FieldName));
         e.Cell.Attributes.Add("fieldName", e.DataColumn.FieldName);
        e.Cell.Attributes.Add("visibleIndex", e.VisibleIndex.ToString());
    }

                                where "onCellDoubleClick" and "onCellClick" are implemented as follows:

                                [JavaScript]
                                        var focusedCell;
        function onCellDoubleClick(ind, fieldName) {
            grid.PerformCallback(ind + "|" + fieldName);
        }
         function onCellClick(index, fieldName) {
            if (focusedCell)
                focusedCell.style.backgroundColor = "white";
            var mainGrid = grid.GetMainElement();
            var allElements = mainGrid.getElementsByClassName("mycell");
            var matchingElements = [];
            for (var i = 0; i < allElements.length; i++) {
                var cell = allElements[i];
                if (cell.getAttribute("visibleIndex") == index) {
                    matchingElements.push(cell);
                }
            }
            var cell;
            for (var i = 0; i < matchingElements.length; i++) {
                cell = matchingElements[i];
                if (cell.getAttribute("fieldName") == fieldName) {
                    cell.style.backgroundColor = "wheat"
                    focusedCell = cell;
                    return;
                }
            }
        }

                                4. Handle the CustomCallback event to change the visibility of other editors and start editing:

                                [C#]
                                    protected void grid_CustomCallback(object sender, DevExpress.Web.ASPxGridViewCustomCallbackEventArgs e) {
        string []parameters = e.Parameters.Split('|');
        foreach (GridViewDataColumn column in grid.Columns) {
            if(column.FieldName != parameters[1]) {
                column.EditFormSettings.Visible = DevExpress.Utils.DefaultBoolean.False;
            } else {
                column.EditFormSettings.Visible = DevExpress.Utils.DefaultBoolean.True;
            }
        }
        grid.StartEdit(Convert.ToInt32(parameters[0]));
    }

                                5. Restrict the user from editing. For this, disable the AllowEdit property based on the user's role as follows:

                                [C#]
                                grid.SettingsDataSecurity.AllowEdit = false;

                                I've attached a sample project and a video to demonstrate how it works. Let me know if you have additional questions.

                                Regards,
                                Paul

                                  Hello Mustafa,

                                  >> when i click reference field "Nationality" in your sample, it opens as text and not popup or dropdown.
                                  >> plus it allows me to insert any random text and click update. and on update it does not saves.

                                  This is because in my example I disabled the IModelView.AllowEdit option in the application model. In this mode, ASPxGridListEditor does not add LookupPropertyEditor's edit cell template to ASPxGridView. However, if you enable this option, an additional command column with the Edit button will be added to ASPxGridView. I did not yet find an appropriate way to hide this column.

                                  >> second, what about validations? it will fire on Update button click , same as on SAVE button click in detail view. 

                                  I cannot guarantee that built-in XAF client validation will work in this scenario, but server-side validation should work properly, because saving is performed by the XAF data access layer.

                                  >> how we can make the edit pop up to open as modal and in center of screen or on top of screen.

                                  I created a separate ticket on your behalf to discuss this requirement: Hot to customize the Edit Form pop-up window.

                                  >> in initial post by paul,  i wanted to ask if my aspxgridview datasource is from multiple tables, then how would i know which table and which field to modify ?

                                  If you are using a built-in XAF ListView, it will automatically save data into appropriate tables using Relationships Between Objects. If you are using a Custom Data Bound Control, you will need to manually handle the RowInserting and RowUpdating events. We have a simple example demonstrating how to use these events to implement a custom data storage layer: ASPxGridView - Editing an in-memory dataset. The concept demonstrated in this example can be used in your scenario.
                                • Uriah (DevExpress Support) 03.06.2018
                                  >> However, if you enable this option, an additional command column with the Edit button will be added to ASPxGridView. I did not yet find an appropriate way to hide this column.

                                  There is a simple workaround that you can use if you do not need to show any command columns. Add the following code to the AdvancedInlineEditController.SetupGrid method:
                                  [C#]
                                  foreach (GridViewColumn column in grid.Columns) {
    GridViewCommandColumn commandColumn = column as GridViewCommandColumn;
    if(commandColumn == null) continue;
    commandColumn.Visible = false;
}
